## JavaScript とはなにか

JavaScript は、インターネットを閲覧するためのソフトウェア（アプリケーション）である「ウェブブラウザ」を中心に、様々な環境上で動作するスクリプト言語です。

JavaScript を自由に記述することができるようになると、最もわかりやすい例としては「ウェブサイト上で動作するプログラム」を自由に実装することができるようになります。また、近年では JavaScript を利用していわゆるデスクトップアプリケーションを作成することもできるようになってきており、JavaScript というたったひとつの言語を覚えるだけで、様々なジャンルのプログラミングに挑戦することができます。

インターネット越しにアクセスできるいわゆる「ウェブサイト」はもちろん、それを配信するための「ウェブサーバー」を構築するのに JavaScript を利用してみたり、あるいはゲームやユーティリティツールのような便利で楽しいアプリケーションを開発することもできます。

さらに言うと、JavaScript で制御・干渉できる領域はいまも現在進行系でどんどん広がっています。近年 JavaScript は非常に人気のあるプログラミング言語になりましたが、そういった背景も人気がある要因のひとつだと言えるでしょう。

> ### スクリプト言語ってプログラミング言語とは違うの？
> 
> まず前提として「スクリプト言語」という言葉に対する明確な基準はありません。
> 
> ○○な要件を満たしているからスクリプト言語だ！ みたいな基準があるわけではない、ということですね。それでは、一般にスクリプト言語と言った場合、これは何を指していると考えればよいのでしょうか。
> 
> 前後の文脈などによって多少の揺れはあると思いますが、スクリプト言語とは「導入や、その実行が、比較的簡易で手間なく行えるもの」を指している場合が多いと言えます。ここで言う「導入や実行」とは、たとえばプログラミング言語を記述するための開発環境の準備や、それを実行して結果を確認するための手順と考えればいいでしょう。
> 
> 実際のところ、スクリプト言語と呼ばれることの多い言語は、比較的簡単に開発環境を整えることができたり、すぐに手間なく（記述したコードの）実行結果を確認できたりするものが多いです。
> 
> ちょっと注意しなければならないのは「スクリプト言語 ＝ 簡単な言語」という意味ではないという点です。環境構築などが比較的容易であるということと、その言語を扱うことが簡単か難しいかということは別の問題ですので、そこを混同しないように注意しましょう。

## サンプルの基本的な構成

前置きが長くなりましたが、さっそく今回のサンプルの内容を見ていきます。

JavaScript 基礎の第１回ということなので、サンプル自体は実にシンプルな構成になっています。

```
0001
 ├ index.html // ブラウザに表示される HTML
 ├ script.js  // index.html から読み込まれるスクリプト
 └ style.css  // index.html から読み込まれる CSS
```

利用しているファイルは３種類あります。

ウェブページとしてウェブブラウザ上に正しく表示されるようにするために `index.html` というファイルを用意し、そこから JavaScript や CSS のファイルを読み込みます。

該当する HTML ファイルの中身は、以下のようになっています。

`<head>` タグの中身を見ると、同階層にある `script.js` や `style.css` を読み込んでいることがわかるかと思います。

</div></section>

ちなみに、ファイル名の手前に `./script.js` のように「ドット＋スラッシュ」が書かれている場合、これはそのファイルが **index.html と同じ階層に置かれている** ことを意味しています。（同じフォルダに入ってるファイルだよ、ということですね）

もし、同じ階層にあるフォルダにファイルが入っている場合などは、その構成に合わせて `./folder/file.js` のように、スラッシュ記号を使って区切ってやれば OK です。

> ### 相対パスと絶対パス
> 
> HTML の中に記述するファイルへの参照や、プログラミングで登場するファイルの位置の指定には、ここで登場したような `./script.js` のようなドットとスラッシュを組み合わせた表記が出てくることがあります。
> 
> まず前提として、プログラミングやパソコン等のコンピューターを扱う分野では、ファイルの位置を指定するこれらのアドレスのことを「パス」と呼びます。そして今回登場したような、ドットとスラッシュを組み合わせたパスの記述は「相対パス」と呼ばれます。なんらかの「起点となるファイルや階層」から見た「相対的な位置を指定するパス」であることから、相対パスと呼ぶわけですね。
> 
> 一方で、相対的な位置ではなく、常に一定の位置を確実に指し示す方法として「絶対パス」という言葉もあります。
> 
> 絶対パスでは、起点となる位置というのがそもそも想定されておらず、常に一定の位置を確実に指し示すために最初から最後まで省略せずにパスを指定します。たとえば `https://webgl.dev/index.html` は絶対パスです。 `https` から始まるすべてのパスが指定されていて、それに完全一致する位置を指し示します。
> 
> HTML にパスを指定する場合、もちろん `https://webgl.dev/sample/0001/script.js` のように絶対パスで記述しても問題ありませんが、ファイルを格納しているフォルダの名前が変わったりすると、その都度パスを修正しなくてはならなくなります。一方で、相対パスで記述されていれば常に相対的な位置でファイルを指定しているため、絶対パスに比較するとパスの修正を必要とするケースは少ないと言えるでしょう。

## JavaScript のコメント

さて、続いては JavaScript 側のコードも見てみましょう。

JavaScript はファイルの拡張子が `*.js` のように js となっているファイルで、HTML 側から `<script>` タグの `src` 属性の指定によって読み込まれると即座に実行されます。

ちなみに、今回のサンプルの場合は `script.js` というファイル名の JavaScript ファイルを読み込んでいますが拡張子以外の部分（ファイル名）は自由に名前を変更しても大丈夫です。（ただしその場合は HTML 側のファイル名の指定も変更しておきましょう）

実際にサンプルの JavaScript ファイルの中身は、以下のようになっています。行数自体は、それほど量は多くないですね。

JavaScript は基本的に「記述されたコードの上から下に向かって順番に」実行されていきます。
</section>

<section class="contentcodesection"><div class="codesectionleft">

```
// インラインコメント
// ※同じ行の、連続するスラッシュ２つ以降をコメントして扱う

/*
    ※複数行をまとめてコメントとして扱う
    alert(1 + 2 + 3 + 4 + 5);
*/

// 開発者ツールのコンソールへの出力
console.log(1 + 2 + 3 + 4 + 5);
```

</div><div class="codesectionright">

JavaScript ではコードの中に **コメント** を記述することができます。

コメントには「その行のなかで、対象の箇所以降をコメント化する」ことができるインラインコメントと、「改行を含む複数の行をまとめてコメント化する」ことができるブロックコメントがあります。

</div></section>

<section class="contenttextsection">

スラッシュを２つ重ねたインラインコメント（ `//` ）では、スラッシュ以降の「同じ行に書かれた内容」がコメントとして扱われます。一方で、スラッシュとアスタリスクを組み合わせて記述するブロックコメントの場合、複数行をまとめてコメント化することも可能です。この場合は `/*` がコメントの開始を意味する記号となり `*/` がブロックコメントの終了を意味する記号となる感じですね。

一般的にプログラミング言語で「コメント」という場合は、プログラムとして実行されない、文字通り単なるコメント（メモ書きや説明）という扱いになります。

プログラムを書いた直後であれば「そのコードが何を意味しているのか、何を行っているのか」は、いましがた書いたばかりのコードですし少なくともその瞬間の本人にとっては明らかな場合が多いでしょう。しかし数日後、数週間後、あるいは数カ月後にそれを見たときには、意味がわからなくなってしまっているなんてこともよく起こります。そういう意味では、コメントを丁寧に残しておく意味はそれなりにあると言えるでしょう。（ただし人により、どの程度コメントを残しておくのが適切と感じられるのか、その感じ方は様々です）

またコメントを書いておく意味・意図としてはその他にも、他人が見てもわかりやすいコードにするためであったり、情報を整理するために特定のルールに則ってコメントを記載しておき、それをドキュメントのように活用するためであったりと、様々な理由があります。

注意しておきたいポイントとしては、コメントがあまりにも多すぎると、逆にコードが読みにくくなってしまったりすることもありますので……ただコメントをたくさん書けばよいと思い込むのではなく、必要に応じて、適切にコメントを残しておけるように努力してみることが大切かなと思います。

## コンソールへの出力

先程掲載したコードのなかに、コメント化されていない部分があったのがわかったでしょうか。

最後のほうに書かれていた `console.log(1 + 2 + 3 + 4 + 5);` の部分は、インラインコメントにも、あるいはブロックコメントにも含まれていませんので、ここは通常どおり JavaScript のコードとして解釈されます。

ここで登場した `console` は、ウェブブラウザに搭載されている **開発者ツール** に対する操作を行うことができるオブジェクトです。この `console` オブジェクトは、JavaScript のプログラミングには欠かせない存在で、主にデバッグ作業やプログラムの動作検証作業などに広く利用されます。

#### 開発者ツールの表示例

![](resource/0001-02.jpg)

今回のサンプルでも利用している `console.log` を利用すると `(` と `)` に包まれた部分に記述された内容が、そのままウェブブラウザの開発者ツールに付属する **コンソール** へと出力されます。計算を行うプログラムが正しく計算を行っているのかを確かめるために計算結果を出力したり、あるいはプログラムの処理内容を確認するためのデバッグ用メッセージを出力したりと、その用途は様々です。

基本的に、コンソールに何を出力するのかは自由なので、プログラムの動作を検証してみたい場合など、適宜コンソールを活用してみるとよいでしょう。

> ### 開発者ツールとコンソール
> 
> 当サイトでは、動作検証を行うブラウザとして Google Chrome を推奨しています。（以降 Chrome と表記）
> 
> Chrome にも、もちろん開発者ツールが付属しています。あらかじめなにか特別なソフトやアプリのインストールなどは一切必要なく、Chrome に初めから備わっており、誰でも自由に利用できます。
> 
> 今回登場した `console` オブジェクトを利用したコンソールへの出力結果は、まずは「開発者ツールを表示する」ことができなければそもそも確認できません。開発者ツールを表示するには、Windows 環境であれば `F12` キーや `Ctrl + Shift + i` の同時押しの操作を行います。Mac 環境の場合は `Command + Option + i` で開発者ツールが表示できます。
> 
> #### 開発者ツールとコンソール
> 
> ![](resource/0001-03.jpg)
> 
> もし、開発者ツールを表示した際にコンソールが表示されていない場合は `Esc` キーを押してみましょう。開発者ツール上にコンソールが表示されると思います。（逆に表示されている状態で `Esc` キーを押すと、コンソールが隠されます）

## アラートの表示

今回のサンプルプログラムのなかで、コメントアウトされている部分（コメント化されていて JavaScript のコードとして実行されない部分）をよく観察してみると、 `alert` という命令が書かれている部分があるのがわかるでしょうか。

該当箇所を抜粋してみると以下のようになります。

#### alert が使われている部分のブロックコメント

```
/*
    ※複数行をまとめてコメントとして扱う
    alert(1 + 2 + 3 + 4 + 5);
*/
```

このブロックコメントを解除してやると、ウェブブラウザの持つ **アラート** の機能が呼び出されるようになります。

ここで言うアラートというのは、いわゆる「ポップアップメッセージ」のことですね。

#### アラートが表示された例

![](resource/0001-04.jpg)

アラートを表示したい場合は `alert` に続く `(` と `)` の間に表示したい内容を指定します。丸括弧の間に出力内容を記述するという意味では、コンソールへの出力と書き方が似ていますね。

開発者ツール上のコンソールは、恐らく「開発者ツールの存在を知っているひと」でなければ目にすることはほとんど無いと思いますが、アラートのほうは画面上にメッセージがポップアップされてくるので、一般ユーザーやウェブサイトを見ている人に対して、よりダイレクトにメッセージを伝えることができます。

たとえば意図したのと異なる値が入力されたとか、時間の掛かる処理が完了したことを知らせる場合など、なにかしらのメッセージをユーザーに伝えたい場合に `alert` が用いられる場合が多いと言えます。

> ### コンソールでお手軽 JavaScript
> 
> Chrome に搭載されているコンソール部分は、実は「即席の JavaScript 実行インターフェース」として利用することができます。
> 
> インターフェースとか言われると、なんかすごく難しいもののように聞こえてしまうかもしれないですが、要するに「JavaScript を実行して結果を確認できる」ということです。
> 
> たとえば、コンソールにフォーカスがある状態（キャレットと呼ばれる縦棒が点滅している状態）で、適当に `alert()` などと入力して `Enter` キーを叩くと、即座にその JavaScript が解釈・実行され、ポップアップメッセージが表示されます。
> 
> 実際に JavaScript を実行して結果を確認してみたい場合など、コンソールへササッとコードを記述して実行させられるというのは大変便利です。たとえば私なんかの場合、ちょっとした計算を行う場合など、電卓ツールを起動するよりブラウザのコンソールで計算してしまうことのほうが多かったりします。ブラウザって大抵開きっぱなしになっていますし、開発の作業を行っている最中であればなおさらコンソールも表示されている場合がほとんどなので、ついついコンソールで計算してしまいます（笑）
